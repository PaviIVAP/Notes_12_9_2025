Feature:
Describes the high-level functionality or module, e.g., “Login Feature”.

Scenario:
Represents a concrete example or test case, e.g., “Login with admin user” or “Login with invalid password”.

Given:
Sets the initial context or preconditions (e.g., user is on login page, user data exists).

When:
Describes the action performed by the user (e.g., clicking login button).

Then:
Defines the expected outcome or assertion (e.g., user sees the dashboard, error message displayed).

And / But:
Used to add multiple steps of the same type for readability and chaining.
